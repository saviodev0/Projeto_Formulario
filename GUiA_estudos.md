# Guia de Estudos

## O que estudar para prova

### 1. Conceitos de frontend

- HTML/CSS/JavaScript básicos
- Componentes React
- Hooks principais: `useState`, `useEffect`
- Formulários controlados e validação
- Consumir APIs com `fetch` ou `axios`
- Ferramentas como Vite para desenvolvimento e build

### 2. Conceitos de backend

- Node.js e Express
- Rotas e endpoints REST
- Middleware Express (`express.json()`, `cors()`)
- Tratamento de erros com `try/catch`
- Envio de respostas JSON
- Uso de banco de dados SQLite com `sqlite3`

### 3. Autenticação e segurança

- Hash de senha com `bcryptjs`
- Validação de entrada no frontend e backend
- Verificar email existente antes de cadastrar
- Não guardar senha em texto plano

### 4. Estrutura do projeto

- Separar frontend e backend em pastas diferentes
- Manter código do servidor isolado de código de interface
- Usar arquivos de configuração adequados
- Documentar scripts e comandos no `README.md`

### 5. Padrões importantes

- `GET`, `POST`, `PUT`, `DELETE` para APIs
- Códigos HTTP corretos: `200`, `201`, `400`, `401`, `409`, `500`
- Respostas com `success: true/false`
- Mensagens claras para o usuário

### 6. Dicas práticas

- Leia sempre as mensagens de erro
- Teste a API com o frontend e com ferramentas como Postman
- Verifique se o servidor está rodando antes de usar o frontend
- Mantenha a leitura de código fácil e comentada

## Prioridade para a prova

1. Saber explicar a diferença entre frontend e backend
2. Entender como o React consome uma API
3. Explicar cada passo do cadastro e login
4. Descrever como funciona a validação de senha
5. Falar sobre o uso de SQLite e bcryptjs
